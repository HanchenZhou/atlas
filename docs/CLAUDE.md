# docs/ 职责说明

本目录是项目的「活文档」——架构、目录结构、关键决策都在这里。
Agent 在 vibe coding 过程中负责保持它的准确性。

> 本文件在工作目录处于 `docs/` 下时由 Claude Code 自动加载；
> 其他位置不会自动加载，但根 `CLAUDE.md` 已包含核心维护触发规则，互相配合即可。

## 文件清单

| 文件 | 职责 | 维护时机 |
|------|------|----------|
| `architecture/background.md` | 项目背景、目标、用户场景、非目标 | 项目方向或目标变化时 |
| `architecture/overview.md` | 整体架构、模块拓扑、数据流、技术选型 | 架构变动时（见根 `CLAUDE.md` 触发条件） |
| `project-structure.md` | 顶层目录结构与各文件夹职责 | 目录变动时（见根 `CLAUDE.md` 触发条件） |

## 维护原则

- **同步更新**：文档改动跟随代码改动，不留 TODO 占位
- **简短优先**：一句话能讲清的事不要写一段
- **链接而不是复制**：引用具体文件路径，不要把代码片段抄进文档
- **删除胜过保留过期内容**：宁可删掉也不留陈旧描述
- **决策记录写"为什么"**：技术选型表格里要写理由，不只是结果

## 不在这里写什么

- API 详细接口文档（应放到代码注释或 OpenAPI 描述里）
- 临时调研笔记（用对话或 commit message）
- 进行中的任务清单（用 TaskCreate 而不是 markdown）
