# 项目背景

## 目标

构建一个支持**知识库检索 + 联网检索**的 agent 应用，具备 agent loop 与工具调用能力，
同时面向**桌面端**与**Web 端**用户提供一致的体验。

## 用户场景

- 用户在本地或云端的知识库中提问，agent 通过检索 + 推理给出答案
- 当本地知识不足时，agent 调用联网搜索工具补充信息
- 桌面端提供本地优先 / 离线友好的体验，Web 端提供随处可用的访问入口

## 关键约束

- **跨端复用**：桌面端与 Web 端共享后端逻辑，不允许双份实现
- **可配置的模型来源**：embedding / LLM 既支持本地（Ollama 等）也支持云 API，由用户选择
- **文档类型不固定**：纯文本、Markdown、PDF、扫描件都可能要支持；复杂解析允许走外部脚本/工具

## 非目标（当前阶段）

- 多用户协同 / 团队空间
- 移动端原生应用
- 自研模型训练 / 微调
- 企业级权限与审计

## 命名

项目代号 **Atlas**——做用户知识的地图与导航。
